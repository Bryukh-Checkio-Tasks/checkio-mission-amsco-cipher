Echemos un vistazo al cifrado “AMSCO”. Este es un algoritmo de cifrado de posición con intercambios. Tu puedes fácilmente usarlo para codificar o decodificar un mensaje utilizando solo lápiz y papel, asumiendo por supuesto, que conoces la clave (key).

La clave es representada como un número cuyos dígitos (únicos) van de 1 a N, siendo N la longitud de la clave. Para codificar un mensaje, debemos escribirlo en una matriz con N columnas. La matriz se escribe fila por fila y en este proceso, uno o dos caracteres se insertan alternativamente en cada celda. Adicionalmente, uno o dos caracteres se alternan en filas y columnas (como un tablero de ajedrez). El primer elemento es el campo de una sola letra (esta es la configuración para esta misión). El último campo puede tener caracteres individuales, si no son suficientes.

Después se numeran las columnas con dígitos de la clave ordenados. Por ejemplo: usando como clave 312, la primera columna será 3, la segunda es 1 y la tercera 2. Por último, se escriben todos los caracteres de las columnas, basándose en la numeración asignada en el paso anterior. Todos los espacios en blanco y signos de puntuación se excluyen, y las letras se escriben en minúsculas.

Veamos esto con un ejemplo. El mensaje de "Lorem ipsum dolor sit amet". Y la clave es 4123. El mensaje de procesado es "loremipsumdolorsitamet". Escrito en forma de matriz seria:

   4   1   2   3
   l  or   e  mi
  ps   u  md   o
   l  or   s  it
  am   e   t

Enseguida se escriben las columnas basándose en la numeración asignada, de forma ascendente - "oruore", "emdst", "mioit", "lpslam". El mensaje codificado es "oruoreemdstmioitlpslam".

Se te asignará un mensaje codificado y la clave. Tu misión es decodificar el mensaje (por supuesto, en la versión procesada).

Datos de Entrada: Dos argumentos. El mensaje como una cadena (unicode) y la clave como entero (int).

Salida: El mensaje descifrado como una cadena (str).

Ejemplo:

decode_amsco("oruoreemdstmioitlpslam", 4123) == "loremipsumdolorsitamet"

¿Cómo se usa?: Para cualquier algoritmo de cifrado, es importante conocer las dos partes - codificación y decodificación. De esta manera, incluso si solo conoces parte del cifrado, es posible realizar una implementación.
